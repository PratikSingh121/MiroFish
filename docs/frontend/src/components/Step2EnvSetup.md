# `frontend/src/components/Step2EnvSetup.vue` — Step 2 Workbench

## Overview

`Step2EnvSetup.vue` drives simulation preparation. It starts the backend prepare task, polls progress, streams partially generated profiles and config, and lets the user either accept the auto-generated round count or override it before moving to the live run.

This component is the UI counterpart of [`services/simulation_manager.py`](../../../backend/app/services/simulation_manager.md).

---

## State and Computed Values

Important refs:
- preparation phase and task identifiers
- progress percentage and stage labels
- `profiles`, `entityTypes`, `expectedTotal`
- `simulationConfig`
- selected profile details
- `useCustomRounds`, `customMaxRounds`
- log container ref

Computed:
- `autoGeneratedRounds`
- `displayProfiles`
- `totalTopicsCount`

---

## Functions

### `getAgentUsername(agentId)`
Looks up a user-facing agent handle.

### `addLog(msg)`
Emits log messages to the parent route.

### `handleStartSimulation()`
Emits the step transition, forwarding either the auto-generated or custom round count.

### `truncateBio(bio)`
Shortens long profile descriptions for card display.

### `selectProfile(profile)`
Opens one profile in the detail panel.

### `startPrepareSimulation()`
Starts backend environment preparation.

### Polling control
- `startPolling()`
- `stopPolling()`
- `startProfilesPolling()`
- `stopProfilesPolling()`
- `startConfigPolling()`
- `stopConfigPolling()`

### Data refresh methods
- `pollPrepareStatus()` reads task progress
- `fetchProfilesRealtime()` streams in-progress profile generation
- `fetchConfigRealtime()` streams in-progress config generation
- `loadPreparedData()` loads finished profiles/config when preparation is already complete
